# プロダクト概要

Google Apps Script を用いて、byTech チームのタスク情報を「Google スプレッドシートを疑似 DB」として保持しつつ、Discord フォーラムチャンネルでの議論と通知を自動化する仕組みです。Workspace 上に配置された本番スプレッドシートを唯一の信頼できるソースと位置づけ、`/Volumes/PortableSSD/Documents/codes/GAS/byTech_Tasks_Managements_Notification/タスク管理シート_開発用.xlsx` はレイアウト検証用サンプルとして参照します。

## コア機能

- **親子タスクの階層管理**: スプレッドシートにはプロジェクト ID を持つ親タスク行があり、紐づく子タスクはタスク ID で管理します。Apps Script はこの階層情報を用いて通知や進捗集計を行います。
- **Discord フォーラム通知**: 各プロジェクト ID ごとに Discord フォーラムスレッドを BOT が作成または既存スレッドへ投稿し、子タスクの状態変化をリアルタイムに共有します。
- **DB 的スプレッドシート連携**: シートは UI ではなくデータストアとして設計し、Apps Script 経由のみで CRUD される前提です。Discord での会話と意思決定が完結するようブリッジします。

## 想定ユースケース

- 新規プロジェクト立ち上げ時に親タスクを登録すると同時に Discord フォーラムにスレッドを自動生成し、以降の議論をそこで管理したいケース。
- 担当者が子タスクの期限更新や完了報告を行った際、対応するスレッドへ即座に通知したいケース。
- 運用チームがスプレッドシートをデータベース的に参照し、ダッシュボードやレポートを生成しながらもコミュニケーションは Discord 内で閉じたいケース。

## 価値提案

- スプレッドシートを DB のみの役割に限定し、ユーザの主導権を Discord に集約することで、チームの作業導線を一本化できます。
- プロジェクト ID / タスク ID の階層ルールにより、通知やスレッドが整理され、親子関係が視覚的にもトレース可能になります。
- 既存の Discord BOT 基盤を活用し、Apps Script 側ではトリガー制御とスプレッドシート操作に専念できるため、保守コストを抑えられます。
